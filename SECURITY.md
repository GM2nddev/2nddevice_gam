# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly by emailing garboy27@hotmail.com.

Do not open a public issue for security vulnerabilities.

## Supported Versions

| Version | Supported |
| ------- | --------- |
| main    | Yes       |

## Security Practices

- All secrets and credentials must be stored in environment variables or a secrets manager, never in code.
- All PRs require review before merging to main.
- Dependency vulnerabilities are monitored via GitHub Dependabot.
